How I use Hugo to build a Website

Posted by HanTing on Sunday, January 19, 2020

現在的發文平台很多，不論會不會程式都能輕易的上手。Facebook、Line、Youtube、IG，它們主要用處還是在於導流。 Blogger、WordPress 是老牌的部落格系統，許多人使用Medium作為部落格系統。Wix 為近期常常在youtube廣告的部落格平台，以及最近誤打誤撞找到的webnote，也都是方便好用的部落格平台

靜態網站產生器 (Static Site Generator) 是輕量級部落格架站的好幫手。 Jekyll、Hexo、Hugo 是目前聽過比較有名的工具。這邊主要使用 Hugo撰寫，目前看到Hugo可以選擇的主題已經夠我去選擇，剩下的就是了解Hugo內部的遊戲規則

工具 语言 優點 備註
Jekyll ruby 主題多、使用者多、Github Pages支援 需要ruby环境
Hexo nodejs 大陸開發的平台，因此中文主題較多 需下載很多nodejs庫，版本更新幅度大
Hugo golang 跨平台、性能强 主題選擇較少，版本更新幅度大

目前使用的部落格主題為 Clean White，會使用它是因為畫面簡單樸素，具有標籤功能與分類功能，功能完善。

使用Clean White的範例：範例1、範例2

安裝

開始前先確認電腦是否有安裝git、npm、VSCode

使用hugo前先安裝hugo，以mac來說執行brew install hugo，其他環境則執行npm install hugo即可

實作

建立文章：hugo new post/post01.md

新增樣式模板：

  • cd themes
  • git clone https://github.com/zhaohuabing/hugo-theme-cleanwhite.git

回到部落格目錄，並啟動 Hugo server hugo server --buildDrafts 瀏覽器中開啟：http://localhost:1313 就能進入首頁。

hugo server 支援 hot reload，任何 html、css、js 修改經儲存後，都能即時顯示。 --buildDrafts （-D）表示草稿也能被看見。

建立html等靜態頁面：只要輸入hugo就會看到 /public 有了可佈署檔案。若輸入hugo --verbose即會在public資料夾內產出對應的html，並印出錯誤訊息

遇到問題

當使用Hugo時會發現一些問題，像是

  • 怎麼沒有查看更多的連結？
  • 要怎麼為自己的文章進行分類？
  • 如何貼出漂亮的code block？

首先針對查看更多的部分，只需加入<!--more-->即可完成，毋需更多設定。

分類文章的部分，詳情請看 Taxonomy

Code block 請參照這裡，而block的css原始碼從這裡下載

Code Block 改動的部份如下程式碼所示

<!-- Use the docco color scheme of highlight.js to highlight the code in the page -->
{{ if .Site.Params.highlightjs }}
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/9.13.1/styles/{{ .Site.Params.highlightjs }}.min.css">
{{ else }}
<link rel="stylesheet" href="/prism.css">
{{ end }}
